SELECT

O comando SELECT usado para selecionar dados de um banco de dados. Os dados retornados so salvos em uma tabela-resultado, chamada result-set.

Sintaxe

SELECT Coluna1, Coluna2, ...
FROM Tabela;

Essa seleo retorna nomes de campos da tabela. Para selecionar todos de uma vez s, usamos a seguinte sintaxe:

SELECT * FROM Tabela;

SELECT DISTINCT

O comando SELECT DISTINCT usado para retornar somente valores estritamente diferentes. Exemplo:

SELECT DISTINCT Coluna1, Coluna2, ...
FROM Tabela

Contagem de valores distintos

O comando SQL a seguir conta e retorna o nmero de valores diferentes em uma tabela:

SELECT COUNT(DISTINCT Country) FROM Customers;

WHERE Clause

A clusula WHERE usada para filtrar a seleo de dados. utilizada para extrair somente dados que atendem uma condio especfica. Exemplo:

SELECT Coluna1, Coluna2, ...
FROM Tabela
WHERE Condicao;

Obs.: A clusula WHERE tambm usada nos comandos UPDATE, DELETE, etc.

Exemplo

Testar para um valor do tipo string

SELECT * FROM Tabela
WHERE Pais = 'Brasil';

Testar para um valor numrico

SELECT * FROM Tabela
WHERE ClienteId = 1;

LIMIT Clause

A clusula LIMIT utilizada para especificar o nmeros de registros a serem retornados. til para quando estamos lidando com uma tabela muito grande, e o nmero de registros pode afetar a performance.

Exemplo

Selecionar os 3 primeiros registros de uma tabela:

SELECT * FROM Customers  
LIMIT 3;

Selecionar os 3 primeiros registros a partir do 3 (no incluso):

SELECT * FROM Customers  
LIMIT 3 OFFSET 3;

Selecionar os 3 primeiros registros junto com uma clusula WHERE:

SELECT * FROM Customers  
WHERE Country='Germany'  
LIMIT 3;

Operadores

Os seguintes operadores podem ser utilizados em uma clusula WHERE:

  • = Equal
  • > Greater than
  • < Less than
  • >= Greater than or equal
  • <= Less than or equal
  • <> Not equal. Nota: Em algumas verses do SQL, esse operador ser !=
  • BETWEEN Between certain range. Ex.: WHERE Price BETWEEN 50 AND 60
  • LIKE Search for a pattern. Ex.: WHERE City LIKE 's%'
  • IN To specify multiple possible values for a column WHERE City IN ('Paris','London')
  • AND,OR,NOT

ORDER BY

O termo-chave ORDER BY usado para ordernar uma tabela-resultado em ordem ascendente ou descendente. Uma tabela-resultado ordenada de forma ascendente por padro; para orden-la de outra forma, utilizamos o termo DESC.

Sintaxe - ordem ascendente, por padro

SELECT * FROM Customers  
ORDER BY Country;

Sintaxe - ordem descendente

SELECT * FROM Customers  
ORDER BY Country DESC;

Ordenar por mais de uma coluna

Ordena primeiramente pela coluna "Country", e se duas linhas tiverem o mesmo valor, ordena em seguida por "CustomerName"

SELECT * FROM Customers  
ORDER BY Country, CustomerName;

Ordena pela primeira coluna de forma ascendente, e pela segunda de forma descendente

SELECT * FROM Customers  
ORDER BY Country ASC, CustomerName DESC;

MIN() and MAX() Functions

A funo MIN() retorna o menor valor da coluna selecionada.
A funo MAX() retorna o maior valor da coluna selecioanda.

Exemplo MIN()

SELECT MIN(Price) AS SmallestPrice  
FROM Products;

Exemplo MAX()

SELECT MAX(Price) AS LargestPrice  
FROM Products;

Obs.: O termo "AS" no cdigo serve para renomear o resultado da consulta. Dessa forma, na sada da consulta a coluna que contm o resultado aparecer como "SmallestPrice", no primeiro caso, ou "LargestPrice", no segundo.
